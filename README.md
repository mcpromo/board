# OREBOARD

Дашборд для учёта продвижения Minecraft-проектов: TikTok/YouTube Shorts, прогресс-бары по просмотрам, тарифы и статистика прибыли. Данные общие — вы и друг видите одни и те же проекты с разных компьютеров, обновления появляются друг у друга сразу.

## Что внутри

- **Статистика вверху**: заработано всего, активные клиенты, всего заказов, всего просмотров — считается автоматически из проектов.
- **Тарифы**: 10K / 400 ₽, 50K / 1450 ₽, 100K / 3750 ₽ — используются при создании проекта, плюс «свой вариант» с произвольной ценой и целью.
- **Проекты**: название, ссылка на видео, прогресс-бар просмотров, кнопка «Обновить просмотры».
- **Напоминание**: если проект не обновлялся сегодня, наверху появляется баннер со списком таких проектов.
- **Общая база данных (Firestore)**: и вы, и друг заходите на один и тот же сайт — данные хранятся в облаке, синхронизация мгновенная.
- Просмотры обновляются вручную: TikTok и YouTube показывают число просмотров прямо на публичной странице видео — просто открываете ссылку и переносите цифру в проект. Отдельного API для этого не нужно.

## Шаг 1. Создать бесплатную базу данных (Firebase)

1. Зайдите на **https://console.firebase.google.com** и войдите с любым Google-аккаунтом.
2. **Add project** → придумайте название (например, `oreboard`) → можно отключить Google Analytics, он не нужен → **Create project**.
3. В меню слева: **Build → Firestore Database → Create database**.
   - Выберите ближайший регион (например, `eur3` для Европы).
   - Режим — **Start in test mode** (это временно, правила поправим на шаге 3).
4. Слева вверху нажмите на ⚙ рядом с "Project Overview" → **Project settings**.
5. Внизу вкладки **General**, в разделе **Your apps**, нажмите иконку **`</>`** (Web).
6. Дайте приложению любое имя → **Register app**. Firebase покажет объект `firebaseConfig` — скопируйте его.
7. Откройте файл **`firebase-config.js`** в проекте и вставьте свои значения вместо `"ВСТАВЬТЕ_СЮДА"`.

## Шаг 2. Задать код доступа (необязательно)

В том же файле `firebase-config.js` есть строка:

```js
const ACCESS_PIN = "1234";
```

Замените `1234` на свой код — это простой экран входа, чтобы случайный человек, узнавший адрес сайта, не сразу попал внутрь. Это **не настоящая защита данных** (сам код сайта всё равно публичный, если кто-то откроет консоль разработчика — PIN не помешает достать данные напрямую из Firebase). Если PIN не нужен вообще — оставьте пустую строку `""`.

## Шаг 3. Настроить правила доступа Firestore

Без авторизации пользователей (её здесь нет — вы просто вдвоём заходите на сайт) нужно явно разрешить чтение и запись. В консоли Firebase: **Firestore Database → Rules**, вставьте:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /projects/{doc} {
      allow read, write: if true;
    }
  }
}
```

Нажмите **Publish**. Это открывает базу для всех, у кого есть ссылка/конфиг сайта — приемлемо для небольшого личного проекта вдвоём с друзьями, но **не подходит**, если вы планируете хранить что-то по-настоящему чувствительное. Если захотите более строгую защиту (например, вход по email/паролю только для вас двоих) — это отдельная доработка, могу её сделать.

## Как загрузить на GitHub

1. Создайте новый репозиторий на GitHub (например, `oreboard`).
2. Загрузите файлы `index.html`, `style.css`, `script.js`, `firebase-config.js`, `README.md`:

   ```bash
   git init
   git add .
   git commit -m "OREBOARD dashboard"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/oreboard.git
   git push -u origin main
   ```

   > ⚠️ Firebase Web API-ключ в `firebase-config.js` — не секретный (он и так виден в браузере любого посетителя сайта), поэтому его можно спокойно класть в публичный репозиторий. Секретность данных обеспечивают именно Firestore Rules из шага 3, а не сокрытие ключа.

3. Включите GitHub Pages: **Settings → Pages** → Branch: `main`, папка `/ (root)` → **Save**.
4. Через минуту-две сайт будет доступен по адресу `https://ВАШ_ЛОГИН.github.io/oreboard/`.
5. Отправьте эту ссылку другу — оба будете работать с одними и теми же проектами.

## Как этим пользоваться

- Нажимаете «+ Новый проект» → вставляете ссылку на TikTok/Shorts, выбираете тариф → проект появляется у вас обоих сразу.
- Раз в день открываете ссылку на каждое видео, смотрите просмотры и вставляете их в «Обновить просмотры» — по одному числу на видео, сумма считается сама, прогресс-бар и статистика наверху пересчитываются автоматически.
- Если проект не обновлялся сегодня — сверху появится напоминание.

## Ссылка для клиента

На каждой карточке проекта есть кнопка **«Ссылка клиенту»** — она копирует персональную ссылку вида `.../?client=ID_ПРОЕКТА`. Откройте её сами один раз, чтобы проверить, и отправьте клиенту.

По этой ссылке клиент видит отдельную страницу: название проекта, процент прогресса и прогресс-бар — без цен, без тарифов, без чужих проектов и без кнопок управления. PIN-код для этой страницы не нужен, но она читает данные напрямую из Firestore по правилам из шага 3, поэтому ссылку стоит выдавать только тем, кому доверяете (случайный человек с этой ссылкой тоже сможет увидеть процент, но не сможет ничего изменить).
